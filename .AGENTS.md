# Agent Notes

This repository manages a personal World of Warcraft addon pack and UI layout.
The current supported target is China TBC Anniversary, while the repository
structure must stay ready for multiple WoW versions.

For a fast handoff in new sessions, read `docs/current-state.md` before making
changes.

## Working Principles

- Keep implementation version-scoped under `src/versions/<version-key>/`.
- Treat `src/versions/tbc-anniversary-cn/layouts/autyan-hunter.v1.json` as the
  source of truth for layout intent.
- Preserve the user's in-game manual adjustments unless explicitly asked to
  overwrite them.
- Prefer small, mature addons with clear responsibilities over large suites.
- Keep Parrot2 as the combat feedback visual anchor.

## Task Flow

Classify work before changing files:

- Addon installation:
  1. Verify the addon version and supported interface numbers.
  2. Install or update files in the target WoW `Interface/AddOns` directory.
  3. Update the character `AddOns.txt`.
  4. Record the addon in `src/versions/tbc-anniversary-cn/addons/manifest.json`.

- Addon configuration:
  1. Check whether WoW is currently running.
  2. If WoW is running, do not write SavedVariables unless the user explicitly
     accepts the risk of the game overwriting those files on exit.
  3. If WoW is closed, write focused SavedVariables changes and keep backups
     when touching existing files.

- Layout work:
  1. Use the layout JSON first.
  2. Calculate abstract layout positions before converting to real screen
     coordinates.
  3. Use the bottom-left coordinate method for action bars and HUD anchors.
  4. Apply plugin-specific coordinates only after the abstract layout is stable.

- Troubleshooting:
  1. Check TOC files, dependencies, enabled state, and interface numbers.
  2. Check SavedVariables only after deciding whether the game is running.
  3. Fix dependency or stale-module issues before changing layout.
  4. Keep unrelated addons and user changes untouched.

## Running Game Constraint

The WoW client can overwrite SavedVariables when it exits. If the game process
is running, file-level configuration changes are not durable by default.

Use process checks before SavedVariables edits, for example:

```sh
pgrep -af 'Wow|WoW|World of Warcraft|WoWClassic|wowclassic' || true
```

## Preferred Division Of Responsibility

- `Bartender4`: action bars.
- `Masque`: button skinning.
- `ShadowedUnitFrames`: unit frames.
- `Parrot2`: combat text and combat feedback.
- `DBM`: encounter alerts.
- `Details + TinyThreat`: damage and threat meters.
- `Questie`: quest guidance.
- `Bagnon`: bag integration.
- `BlizzMove`: movable Blizzard windows.
- `Skinner`: Blizzard window skinning.
- `OmniCC + ClassicAuraDurations`: cooldown and aura duration text.
- `AtlasLootClassic`: loot/source browsing.
- `VendorPrice`: sell price tooltip.
- `Dejunk`: auto-sell workflow.
- `Feed_Me`: hunter pet feeding helper.

Keep tooltip information and auto-selling separate: tooltip addons explain the
item, while Dejunk decides what to sell.
